# 开发提示词模板

## 提示词目标
将设计稿分析结果转换为详细的开发指令，指导 AI 生成符合设计稿的前端代码。

## 提示词结构

### 1. 项目概述
- 技术栈选择（React + TypeScript + Vite + Ant Design）
- 项目目标描述

### 2. 整体布局
- 页面整体结构描述
- 各区域尺寸和位置
- 背景颜色和样式

### 3. 功能模块详细要求

#### 交互设计细节
对于每个模块，需描述以下交互状态（若适用）：
- **Hover 状态**：鼠标悬停时的背景色、边框或文字颜色变化。
- **Focus 状态**：输入框或按钮聚焦时的边框颜色、阴影效果。
- **Active/Click 状态**：点击时的按压效果或加载状态（Loading）。
- **Empty/Error 状态**：无数据或加载失败时的占位图或提示文字。

#### 顶部导航栏
- 左侧区域详细说明
- 中间区域详细说明
- 右侧区域详细说明

#### 左侧导航菜单
- 样式规范
- 菜单项列表
- 交互状态

#### 其他功能模块
- 按模块逐一详细说明

### 4. 样式规范总结

#### 色彩方案
- 所有颜色值及用途表格

#### 字体层级
- 各级字体大小、粗细、颜色

#### 组件样式
- 按钮、输入框、标签等组件样式

#### 间距规范
- 各种间距值

## 提示词编写原则

### 必须包含的信息
- 每个元素的具体位置
- 每个文本的精确内容
- 每个样式的精确值（颜色、尺寸等）
- 组件之间的层级关系
- 交互状态的描述

### 推荐包含的信息
- 组件 props 定义
- 状态管理方式
- Mock 数据结构
- 路由配置

## 示例格式

```markdown
# [项目名称] - [页面名称] 开发提示词

## 项目概述
请使用 [技术栈] 开发一个 [项目描述]。

## 整体布局
- [布局描述]

## 功能模块详细要求

### 1. [模块名称]
[详细说明]

### 2. [模块名称]
[详细说明]

## 样式规范总结

### 色彩方案
| 用途 | 颜色值 | 说明 |
|------|--------|------|
| [用途] | [颜色值] | [说明] |

### 字体层级
- [层级描述]

### 组件样式
- [样式描述]

### 间距规范
- [规范描述]
```

## 注意事项

1. **完整性**：提示词应包含所有必要的开发信息
2. **精确性**：颜色、尺寸等数值必须精确
3. **结构化**：使用清晰的层级结构
4. **可操作性**：提示词应能直接用于代码生成
