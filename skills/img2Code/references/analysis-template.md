# 设计稿分析模板

## 分析目标
将设计稿中的所有视觉元素、布局、样式、交互等细节整理成结构化文档，为后续代码生成提供完整参考。

## 分析维度

### 1. 整体布局分析
- 页面整体结构（如：左侧菜单 + 主内容区）
- 各区域尺寸和位置
- 背景颜色和层级关系
- 响应式布局特征

### 2. 顶部导航栏分析
- Logo 区域（图标形状、文字、背景）
- 面包屑导航（图标、文字、链接样式）
- 右侧功能区（下拉框、通知、用户信息）
- 分隔线和边框

### 3. 侧边栏菜单分析
- 菜单项列表（名称、图标、顺序）
- 选中状态样式
- 未选中状态样式
- 底部功能区

### 4. 主内容区分析
- 页面标题和描述
- 各功能模块划分
- 模块间关系和布局

### 5. 组件详细分析
对每个组件记录：
- 组件名称和位置
- 包含的元素
- 文本内容（精确到每个字符）
- 样式属性（颜色、字体、圆角、阴影等）
- 交互状态（hover、active、focus 等）
- 数据和内容

### 6. 样式规范总结
- 色彩方案（所有颜色值及用途）
- 字体层级（大小、粗细、颜色）
- 组件样式（按钮、输入框、标签等）
- 间距规范（内边距、外边距、间隙）

## 输出格式要求

### 文档结构
```markdown
# [项目名称] - [页面名称] 设计稿分析

## 设计稿概览
[整体描述]

## 功能模块分析

### 一、[模块名称]
**布局**：[布局描述]
**内容**：
| 位置 | 元素 | 样式/交互 |
|------|------|-----------|
| ... | ... | ... |

[按模块逐一分析]

## 整体样式总结

### 色彩方案
| 用途 | 颜色 | 说明 |
|------|------|------|
| ... | ... | ... |

### 字体层级
- [字体层级列表]

### 组件样式
- [组件样式列表]

### 间距规范
- [间距规范列表]

## 页面布局结构
[ASCII 布局图]
```

## 检查清单

### 必须包含的内容
- [ ] 所有功能模块已识别
- [ ] 每个模块的文本内容已记录
- [ ] 样式属性（颜色、字体、圆角等）已记录
- [ ] 交互状态已说明
- [ ] 布局关系已描述
- [ ] 数据内容已记录

### 可选但建议包含的内容
- [ ] 组件层级关系
- [ ] 响应式布局特征
- [ ] 动画效果描述
- [ ] 无障碍访问考虑

## 注意事项

1. **精确性**：文本内容必须与设计稿完全一致
2. **完整性**：不遗漏任何可见元素
3. **结构化**：使用表格、列表等结构化格式
4. **可复用**：分析结果应能直接用于代码生成
